測定メモ(15/12/24:akita11)
入力: a, b, c, d
出力: q0 - q15
仕様：4ビット入力→16ビット出力のデコーダ

測定方法： a, b, c, dをArduinoで16通りの組み合わせを生成、
q0 - q15とともにロジックアナライザで波形を計測

結果：151224Maruyama{1,2}.jpg

考察：
入出力の対応がおかしいように見える。
q0 - q15が"1"となるのは、a - dが以下の場合。(カッコ内が順にd, c, baの値)

q0:  (1, 1, 1, 1)
q1:  (0, 1, 1, 1), (1, 1, 1, 1)
q2:  (1, 0, 1, 1), (1, 1, 1, 1)
q3:  (0, 0, 1, 1), (1, 1, 1, 1)
q4:  (1, 1, 0, 1), (1, 1, 1, 1)
q5:  (0, 1, 0, 1), (1, 1, 1, 1)
q6:  (1, 0, 0, 1), (1, 1, 1, 1)
q7:  (0, 0, 0, 1), (1, 1, 1, 1)
q8:  (1, 1, 1, 0), (1, 1, 1, 1)
q9:  (0, 1, 1, 0), (1, 1, 1, 1)
q10: (1, 0, 1, 0), (1, 1, 1, 1)
q11: (0, 0, 1, 0), (1, 1, 1, 1)
q12: (1, 1, 0, 0), (1, 1, 1, 1)
q13: (0, 1, 0, 0), (1, 1, 1, 1)
q14: (1, 0, 0, 0), (1, 1, 1, 1)
q15: (0, 0, 0, 0), (1, 1, 1, 1)

ただし、a→/d, b→/c, c→/b, d→/aと変換すると、以下となる。(/はNOT演算)
q0:  (0, 0, 0, 0)
q1:  (0, 0, 0, 1), (0, 0, 0, 0)
q2:  (0, 0, 1, 0), (0, 0, 0, 0)
q3:  (0, 0, 1, 1), (0, 0, 0, 0)
q4:  (0, 1, 0, 0), (0, 0, 0, 0)
q5:  (0, 1, 0, 1), (0, 0, 0, 0)
q6:  (0, 1, 1, 0), (0, 0, 0, 0)
q7:  (0, 1, 1, 1), (0, 0, 0, 0)
q8:  (1, 0, 0, 0), (0, 0, 0, 0)
q9:  (1, 0, 0, 1), (0, 0, 0, 0)
q10: (1, 0, 1, 0), (0, 0, 0, 0)
q11: (1, 0, 1, 1), (0, 0, 0, 0)
q12: (1, 1, 0, 0), (0, 0, 0, 0)
q13: (1, 1, 0, 1), (0, 0, 0, 0)
q14: (1, 1, 1, 0), (0, 0, 0, 0)
q15: (1, 1, 1, 1), (0, 0, 0, 0)
これは、(0, 0, 0, 0)の場合にすべてのq0-q15がHとなる、こと以外は、デコーダとして正しい挙動といえる。
